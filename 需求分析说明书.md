# 空气质量数据可视化分析平台 - 需求分析说明书

## 1. 引言

### 1.1 编写目的
本文档旨在明确空气质量数据可视化分析平台的功能需求、业务流程和系统边界，为后续开发提供依据。

### 1.2 项目背景
随着环境污染问题日益受到关注，公众对空气质量信息的需求不断增加。本平台通过自动化采集多源数据，构建一个集中、直观的空气质量信息中心，为环境感知与健康出行提供数据支持。

### 1.3 术语定义
| 术语 | 说明 |
|------|------|
| AQI | 空气质量指数(Air Quality Index)，数值越大表示空气污染越严重 |
| PM2.5 | 细颗粒物，直径小于等于2.5微米的颗粒物 |
| PM10 | 可吸入颗粒物，直径小于等于10微米的颗粒物 |
| ARIMA | 自回归积分滑动平均模型，用于时间序列预测 |

## 2. 系统概述

### 2.1 系统目标
- 自动采集全国重点城市空气质量数据
- 提供实时空气质量查询与可视化展示
- 支持历史数据趋势分析与对比
- 实现空气质量预警功能

### 2.2 系统范围
**包含功能：**
- 数据采集与存储
- 实时数据展示（地图、排名）
- 历史数据查询与趋势分析
- 多城市数据对比
- 空气质量预警

**不包含功能：**
- 用户注册登录系统（当前版本为公开访问）
- 移动端APP（当前为Web端）
- 数据导出功能

### 2.3 运行环境
- **服务端**：Java 8 + SpringBoot + MySQL 8.0
- **客户端**：Chrome/Firefox/Edge 等现代浏览器
- **采集端**：Python 3.8+

## 8. 数据需求

### 8.1 数据实体

**城市信息(City)**
- 城市代码(city_code)：主键，如"110000"
- 城市名称(city_name)：如"北京"
- 省份(province)
- 经度(longitude)、纬度(latitude)

**空气质量历史数据(AirQuality)**
- 城市代码、日期、AQI、空气质量等级
- PM2.5、PM10、SO2、NO2、CO、O3浓度

**空气质量实时数据(AirQualityRealtime)**
- 城市代码、AQI、各污染物浓度
- 数据时间

**预警记录(WarningRecord)**
- 城市代码、预警日期、预测AQI
- 预警等级、预警信息、处理状态

### 8.2 数据量估算
- 城市数量：20个
- 采集频率：每小时1次
- 日数据量：20条实时 + 20条历史 = 40条
- 年数据量：约1.5万条历史数据

## 10. 附录

### 10.1 AQI等级划分标准

| AQI范围 | 等级 | 颜色 | 健康影响 |
|---------|------|------|----------|
| 0-50 | 优 | 绿色 | 空气质量令人满意 |
| 51-100 | 良 | 黄色 | 空气质量可接受 |
| 101-150 | 轻度污染 | 橙色 | 敏感人群症状轻度加剧 |
| 151-200 | 中度污染 | 红色 | 进一步加剧易感人群症状 |
| 201-300 | 重度污染 | 紫色 | 心脏病和肺病患者症状显著加剧 |
| >300 | 严重污染 | 褐红色 | 健康人群运动耐受力降低 |

### 10.2 监测城市列表

北京、上海、广州、深圳、杭州、南京、武汉、成都、重庆、天津、西安、郑州、长沙、济南、石家庄、沈阳、哈尔滨、长春、苏州、宁波
