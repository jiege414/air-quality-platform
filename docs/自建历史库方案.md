# 自建历史库方案实施指南

## 方案概述

通过持续运行爬虫调度器，每小时抓取 PM25.im 数据并存储到历史表，逐步积累历史数据。

## 时间线

- **1 周后**：可查询近一周数据
- **1 月后**：可查询近一月数据  
- **3 月后**：可查询近三月数据
- **1 年后**：可查询近一年数据

---

### 第 2 步：修改爬虫代码

修改 `pm25im_spider.py`，在保存实时数据的同时，也保存到历史表：

```python
def save_to_database(self, data_list):
    """保存数据到数据库（实时表 + 历史表）"""
    if not data_list:
        logger.warning("没有数据需要保存")
        return 0
    
    success_count = 0
    
    try:
        # 1. 清空并更新实时数据表
        db.execute("DELETE FROM air_quality_realtime")
        
        # 2. 插入新数据到实时表
        sql_realtime = """
            INSERT INTO air_quality_realtime 
            (city_code, city_name, aqi, quality, pm25, pm10, so2, no2, co, o3, data_time)
            VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
        """
        
        # 3. 同时插入历史表（保留历史）
        sql_history = """
            INSERT INTO air_quality_history 
            (city_code, city_name, aqi, quality, pm25, pm10, so2, no2, co, o3, data_time)
            VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
        """
        
        for data in data_list:
            params = (
                data['city_code'], data['city_name'], data['aqi'],
                data['quality'], data['pm25'], data.get('pm10', 0),
                data.get('so2', 0), data.get('no2', 0),
                data.get('co', 0), data.get('o3', 0), data['data_time']
            )
            
            # 保存到实时表
            db.execute(sql_realtime, params)
            
            # 保存到历史表
            db.execute(sql_history, params)
            success_count += 1
        
        logger.info(f"成功保存 {success_count} 条数据到实时表和历史表")
        return success_count
        
    except Exception as e:
        logger.error(f"数据库操作失败: {str(e)}")
        return 0
```

### 第 3 步：部署运行方式

你有以下几种运行方式：

#### 方式 A：本地电脑持续运行（简单但不稳定）

```bash
# 在 spider 目录下运行
python scheduler.py
```

**缺点**：
- 电脑关机就停止
- 不适合长期运行

#### 方式 B：使用云服务器（推荐）

购买一个云服务器（阿里云/腾讯云/华为云），24小时运行：

```bash
# 1. 购买云服务器（最低配即可，约 50-100元/月）

# 2. 部署项目到服务器

# 3. 使用 systemd 或 supervisor 守护进程
```

#### 方式 C：使用定时任务（折中方案）

如果你不想一直开着电脑，可以用 Windows 计划任务：

```
每天运行 4 次：
- 08:00 运行一次
- 12:00 运行一次
- 16:00 运行一次
- 20:00 运行一次
```

这样一天抓取 4 次，虽然不够密集，但也能积累历史数据。

## 运行要求

### 硬件要求

| 配置 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 1核 | 2核 |
| 内存 | 1GB | 2GB |
| 硬盘 | 10GB | 20GB |
| 网络 | 1Mbps | 2Mbps |

### 存储估算

每小时存储 20 个城市的数据：

- 每天：20 条/小时 × 24 小时 = 480 条
- 每月：480 条 × 30 天 = 14,400 条
- 每年：14,400 条 × 12 月 = 172,800 条

数据量很小，普通数据库完全可以承受。
