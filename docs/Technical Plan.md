### 方案一：和风天气 API（推荐）
##失败原因：免费版不能获取aqi数据
实现步骤：

1. 注册账号
   
   - 访问 https://dev.qweather.com/
   - 注册开发者账号
   - 创建应用获取 API Key
2. API 调用方式
   
   ```
   GET https://devapi.qweather.com/v7/air/now?
   location=101010100&key=YOUR_KEY
   ```
3. 城市代码映射
   
   - 和风天气使用数字城市代码（如北京：101010100）
   - 需要建立城市名称 ↔ 城市代码的映射表
4. 数据转换
   
   - 将和风天气返回的 JSON 格式转换为项目数据库格式
优点：

- 数据真实可靠
- 接口稳定
- 免费额度足够
缺点：

- 需要注册账号
- 需要维护城市代码映射

### 方案二：OpenWeatherMap（备选）
##失败原因：数据异常偏高（可能是国外的计算方式和我们不一样）
实现步骤：

1. 注册账号
   
   - 访问 https://openweathermap.org/
   - 注册获取 API Key
2. API 调用
   
   ```
   GET http://api.openweathermap.org/data/2.5/
   air_pollution?lat=39.9042&lon=116.4074&appid=YOUR_KEY
   ```
3. 坐标转换
   
   - 使用城市经纬度查询
   - 需要维护城市坐标数据
优点：

- 全球覆盖
- 国际知名度高
缺点：

- 国内数据精度不如和风天气
- 需要处理坐标转换
### 方案三：网页爬取兜底（cnemc）
实现思路：

1. 分析网页结构
   
   - 使用浏览器开发者工具分析 cnemc.cn 的数据加载方式
   - 寻找 AJAX 接口或隐藏 API
2. 反爬虫对抗
   
   - 设置合理的请求头（User-Agent、Referer）
   - 使用代理 IP 池轮换
   - 控制请求频率（每 2 小时一次）
   - 添加随机延迟
3. 数据解析
   
   - 使用 BeautifulSoup 或正则表达式提取数据
   - 处理 JavaScript 动态渲染（可能需要 Selenium）
优点：

- 数据最权威
- 无调用限制
缺点：

- 实现复杂
- 容易被封 IP
- 需要持续维护（网站改版会失效）
